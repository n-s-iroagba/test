PERSONA: MARCUS (THE INTERNATIONAL SCOUT)
1. Discover & Register for Pro View Portal
Goal: Gain exclusive access to verified player data and match archives.
1. Navigates to website.
2. Finds "Scout/Pro View Access" CTA.

Goal: Assess player potential using reliable, audited data.
1. Logs into Pro View portal.
2. Browses/searches for a player.
3. Clicks to view detailed profile (Name, Image, DOB, Height, Position, Bio).
4. (Optional) Views audit trail for profile edit history.

3. Analyze Performance via Match Archives
Goal: Watch full matches to scout performance.
1. Logs into Pro View portal.
2. Navigates to "Match Archives."
3. Selects a fixture.
4. Accesses embedded full-match video (YouTube/Vimeo).
5. Video loads with adaptive bitrate for smooth playback.
4. Export Player Data for External Analysis
Goal: Get player list for offline review.
1. Logs into Pro View portal.
2. Navigates to player list/search.
3. Applies filters (e.g., Position, Age).
4. Clicks "Export to CSV."
5. Downloads CSV (Name, Position, Age).
5. Attempt Early Access to Match Archives
Goal: Access archives before processing is complete.
1. Logs in shortly after match ends.
2. Navigates to new match in archives.
3. Finds archives unavailable if <30 minutes post-match.
4. Returns later to find them available.
6. Experience Content Segregation
Goal: Confirm exclusive access to Pro View content.
1. Logs into Pro View portal.
2. Browses available content.
3. Verifies they cannot see draft or admin-only content areas.

PERSONA: FATIMA (THE LOCAL BUSINESS OWNER / ADVERTISER)
1. Register as a Verified Advertiser
Goal: Create an account to start advertising.
1. Clicks "Advertise" CTA on website.
2. Fills registration form (Business Name, Email, Phone).
3. Submits form; account status becomes "Pending Verification."
4. Receives approval email from Commercial Manager.
5. Logs into advertiser dashboard.
2. Create & Launch an Ad Campaign
Goal: Set up targeted ad campaign and pay.
1. Logs into advertiser dashboard.
2. Clicks "Create New Campaign."
3. Selects ad zone (sees exact specs/price).
4. Uploads creative; system validates dimensions/format/size.
5. Creative auto-tagged with "Ad by [Advertiser], [Company]".
6. Sets targeting (Home Page OR specific content tags).
7. Defines target unique views.
8. System calculates total cost (per-view rate × target).
9. Proceeds to Paystack checkout; completes payment.
10. Receives payment confirmation email.
11. Campaign becomes "Active."
3. Monitor Campaign Performance
Goal: Check real-time ad delivery metrics.
1. Logs into advertiser dashboard.
2. Views campaign list/performance dashboard.
3. Sees metrics: Purchased, Delivered, Remaining views, CPV.
4. Data updates with ≤5 minute latency.
4. Respond to Campaign Completion
Goal: Be notified when campaign finishes.
1. Receives email: campaign reached 100% delivery, auto-paused.
2. Email includes upsell option for new campaign.
3. Logs in to see paused campaign at 100% delivered.
5. File a Delivery Dispute
Goal: Report suspected ad delivery issue.
1. Sends email to complaints@amaforgaladiatorsfc.com.
2. Receives auto-acknowledgement with case number.
3. Receives substantive response from Commercial Manager within 2 business days.
6. Receive Ad Rate Change Notification
Goal: Be informed of pricing changes.
1. Receives email 30 days prior to per-view rate change.
2. Email details new rates and effective date.
7. Attempt to Advertise During Payment Outage
Goal: Create campaign when Paystack is down.
1. Tries to create campaign/reach payment.
2. Sees system maintenance message.
3. Is prevented from completing purchase.
8. View & Export Historical Performance Data
Goal: Analyze past campaign data.
1. Logs into dashboard.
2. Navigates to historical reports (data retained for 1 year).
3. (Optional) Exports campaign data (Campaign, Views, Spend) to CSV.
9. Confirm Data Segregation
Goal: Verify privacy of advertiser data.
1. Logs into dashboard.
2. Browses campaign/list views.
3. Confirms they cannot see other advertisers' data/campaigns.
10. Handle Upload Validation Failure
Goal: Correct a failed creative upload.
1. Attempts to upload invalid creative (wrong size/format).
2. Receives clear error message stating the reason.
3. Corrects the file and re-uploads successfully.
11. Verify Ad Targeting
Goal: Confirm ad appears on correct pages.
1. Creates campaign targeting specific tags (e.g., "Football News").
2. After approval, visits pages tagged "Football News."
3. Confirms their ad appears on those pages.

SYSTEM-DRIVEN / AUTOMATED JOURNEYS
1. Auto-Pause Ad Campaign at 100% Delivery
1. Campaign runs, tracking unique views.
2. Delivered views reach 100% of purchased target.
3. (System) Automatically changes campaign status to "Paused."
4. (System) Sends email notification to advertiser with upsell option.
2. Auto-Acknowledge Complaint Email
1. Email sent to complaints@amaforgaladiatorsfc.com.
2. (System) Detects new email to this address.
3. (System) Sends auto-reply with case number.
4. (System) Logs complaint in tracking system.
3. Hourly Sync of Ad Views from GA4
1. Google Analytics records GA4 event for an ad view.
2. (System) Hourly cron job queries GA4 for new events.
3. (System) Processes events, updates internal ad delivery counters.
4. Dashboard data reflects updates with ≤5 min latency from sync.
4. Log Audit Trail on Data Change
1. User with edit privileges changes a field in an audited table (e.g., Player Profile).
2. (System) Intercepts the save operation.
3. (System) Records Old Value, New Value, User ID, Timestamp to immutable log table.
5. Insert Mid-Article Ad
1. User views an article page.
2. (System) Parses article content, counts ~100 words.
3. (System) Inserts a Mid-Article ad zone (640x360 video) after the 100th word.
4. If article <100 words, ad is inserted at the end.
6. Track an "Ad View"
1. User's browser loads a page with an active ad.
2. Ad becomes 50% visible in viewport for ≥1 second.
3. (System) Frontend script triggers a GA4 event for a "view."
4. (System) Sets/checks a persistent cookie to count only once per user per 24h.
5. Event is logged for hourly sync (Journey S.3).
7. Enforce SSL/TLS Across Site
1. User or system attempts HTTP access to any page/API.
2. (System) Redirects request to HTTPS equivalent.
3. All data transmission is encrypted.
8. Perform Automated Database Backups
1. (System) Scheduled job runs (e.g., daily).
2. Creates backup snapshot of database.
3. Stores backup in secure, off-site location.
4. (Admin) Can later verify restore procedure from backup.
9. External Uptime Monitoring
1. (External Service) Pings public website at regular intervals.
2. If ping fails/response slow, triggers alert to DevOps/Admin.
3. Tracks uptime percentage against 99.5% target.
10. Send Scheduled Rate Change Emails
1. Admin changes an ad rate, effective on a future date.
2. (System) Calculates date 30 days prior to effective date.
3. On that date, (System) emails all advertisers on record about the upcoming change.

COMPLIANCE, SECURITY & VALIDATION JOURNEYS
1. Unauthorized Access Attempt
1. User tries to directly access /admin or /pro-view URL without logging in.
2. (System) Redirects user to login page.
2. Access Denied for Low-Privilege User
1. User with "Media Manager" role tries to access /admin/financial-reports.
2. (System) Returns "403 Forbidden" or similar access-denied page.
3. Validate WCAG 2.1 AA Compliance
1. (QA/Auditor) Runs axe DevTools or similar on key pages.
2. Reviews report for violations of contrast, alt text, keyboard nav, etc.
3. Confirms compliance or logs issues for fix.
4. Simulate View Count Spam Attack
1. (QA/Tester) Uses script to rapidly refresh a page with an ad/article.
2. (QA/Tester) Checks internal logs/database.
3. Confirms view counts increased only once per valid view, resisting spam.
5. Validate ISO 27001 Control Mapping
1. (Auditor) Reviews system architecture, access logs, backup procedures, audit trails.
2. Maps each observed control to the ISO 27001:2022 framework.
3. Produces compliance report.
6. Load Test Search Performance
1. (QA/Dev) Populates database with 10,000+ test records (players, articles).
2. Uses load testing tool to simulate multiple concurrent search queries.
3. Confirms search response time remains <2 seconds under load.
7. Perform Accessibility Audit
1. Follows Journey C.3 (Validate WCAG Compliance).
2. Also manually tests with screen readers and keyboard-only navigation.
8. Data Subject Access Request (Implied)
1. User requests a copy of their personal data.
2. (Admin) Uses admin tools to compile data associated with user's email/ID.
3. Provides secure download link to user.
9. Data Purge Request/Retention
1. User requests account/data deletion.
2. OR, data retention period expires.
3. (Admin/System) Anonymizes/purges user data from system in accordance with policy.
